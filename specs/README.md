# Specs

Everything under `specs/` is **design intent**: plans, explorations, and target
designs frozen at writing time — not a description of current behavior. The
implemented API and current behavior are documented under `docs/`; when a spec
and `docs/` disagree, trust `docs/`. Never cite a spec as evidence of what the
code does today.

- `reviewed/` — design specs that went through review (2026-05); each carries a design-intent status banner
- `not_reviewed/` — drafts and explorations
- `deprecated/` — superseded designs kept for history
- `examples/`, `references/` — comparison material from other frameworks
